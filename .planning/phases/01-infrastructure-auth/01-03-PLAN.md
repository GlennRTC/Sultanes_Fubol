<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet -> ping Edge Function | Unauthenticated callers can hit ping (--no-verify-jwt) |
| build pipeline -> bundled client | VITE_ env vars are embedded in public JS by design |
| Netlify CDN -> SPA | Deep-link routing handled by the catch-all redirect |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-11 | Denial of Service | public ping function abuse | accept | ping is stateless, touches no DB, and returns a tiny JSON; Supabase free-tier invocation cap (500k) is far above a 3-day cron; no amplification surface |
| T-01-12 | Information Disclosure | secrets in client bundle | mitigate | Only VITE_SUPABASE_ANON_KEY + URL are set in Netlify env; service role key and football API key are never referenced in frontend code (CLAUDE.md) |
| T-01-13 | Tampering | ping function as a write vector | mitigate | ping imports no Supabase client and performs no DB writes — it cannot mutate state even though it is public |
| T-01-14 | Information Disclosure | data loss without backups | mitigate | INF-06 daily backups confirmed enabled in the Supabase dashboard |
| T-01-SC | Tampering | supabase/netlify CLI installs | mitigate | CLIs are official first-party tools (Supabase, Netlify); no application npm dependencies added in this plan |
</threat_model>
