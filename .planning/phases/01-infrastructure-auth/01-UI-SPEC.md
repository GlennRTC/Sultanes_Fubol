# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Infrastructure + Auth.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> All UI copy in Spanish. Code in English.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (TailwindCSS utility-first — fixed stack constraint) |
| Preset | not applicable |
| Component library | none — hand-built components with Tailwind classes |
| Icon library | none in Phase 1 — use text labels and Unicode symbols only |
| Font | System UI stack: `ui-sans-serif, system-ui, -apple-system, sans-serif` |

**Rationale (shadcn gate):** `components.json` not found. shadcn was evaluated but declined for Phase 1 because: (1) TailwindCSS is already the fixed stack choice; (2) 7-day timeline to June 11 launch makes setup overhead a liability; (3) Phase 1 auth components are simple enough to build without a component registry. The patterns established here become the baseline for Phases 2–4. If shadcn is desired, initialize in Phase 4 during the QA/polish phase.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label-to-field gaps |
| sm | 8px | Input internal padding (vertical), button text gap |
| md | 16px | Input internal padding (horizontal), card padding, form field gap |
| lg | 24px | Section padding, form group gap |
| xl | 32px | Auth card padding (desktop), page vertical padding |
| 2xl | 48px | Auth card top/bottom margin on mobile |
| 3xl | 64px | Full-screen container centering offset |

Exceptions:
- Touch targets for the "Cerrar sesión" button and nav elements: minimum 44px height (accessibility, iOS Safari tap target)
- Full-screen auth spinner: fills 100vh / 100vw

## Typography

All type is set in the system UI stack declared above. No Google Fonts or external font load in Phase 1 — preserves free-tier bandwidth.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 700 (bold) | 1.2 |
| Display | 28px | 700 (bold) | 1.1 |

**Declared weights: 400 (regular) and 700 (bold) only. No third weight.**

**Usage notes:**
- Body (16px/400): form field values, nav username, paragraph copy
- Label (14px/400): form field labels, helper text, crosslink text ("¿No tienes cuenta?")
- Heading (20px/700): page titles ("Iniciar sesión", "Crear cuenta"), section headers
- Display (28px/700): "FUBOL" logo text in the navbar

## Color

Football World Cup palette — dark navy dominant with green accent. All values must pass WCAG AA (4.5:1 contrast for text on background).

| Role | Tailwind Class | Hex Approx | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | `bg-slate-900` | #0f172a | Page background, full-screen spinner background |
| Secondary (30%) | `bg-slate-800` | #1e293b | Navbar background, auth card background |
| Accent (10%) | `bg-green-500` | #22c55e | Reserved-for list below |
| Destructive | `bg-red-600` | #dc2626 | Error text, destructive action button (none in Phase 1) |

**Accent (`green-500`) reserved for:**
1. Primary CTA button background: "Iniciar sesión" and "Crear cuenta" submit buttons
2. "Fichas: 0" token balance label in the navbar (text color only: `text-green-400`)
3. Success message text on `/login` after password reset ("Contraseña actualizada. Inicia sesión.")
4. Input focus ring: `focus:border-green-500 focus:ring-green-500` on all `<FormField>` inputs

**Text colors:**
- Primary text on dark: `text-slate-100` (#f1f5f9)
- Secondary / muted text: `text-slate-400` (#94a3b8)
- Accent text (token balance, success): `text-green-400` (#4ade80)
- Error text: `text-red-400` (#f87171)
- Link / crosslink text: `text-green-400 hover:text-green-300`

**Border:**
- Form field border: `border-slate-600` default, `border-green-500` on focus
- Card border: `border-slate-700`

## Component Inventory

Components introduced in this phase. All hand-built with TailwindCSS.

### `<Navbar>`
- Height: 56px (min-h-[56px] or h-14)
- Background: `bg-slate-800 border-b border-slate-700`
- Left: "FUBOL" in Display type (28px/700, `text-white`)
- Right (authenticated): `[username]` (Body, `text-slate-100`) + `Fichas: [n]` (Label, `text-green-400`) + "Cerrar sesión" button
- "Cerrar sesión": plain text button, Label size, `text-slate-300 hover:text-white`, no background, min-h-[44px]
- Right (unauthenticated): hidden — navbar not shown on `/login` or `/registro`

### `<AuthCard>`
- Max width: 400px, horizontally centered
- Padding: 32px (xl) on all sides (desktop); 24px (lg) on mobile
- Background: `bg-slate-800`, border: `border border-slate-700`, border-radius: `rounded-xl`
- Vertical margin: auto (centered vertically in remaining viewport height)
- Heading at top: Heading type (20px/700, `text-slate-100`)

### `<FormField>` (label + input wrapper)
- Label: 14px/400, `text-slate-300`, margin-bottom: 4px (xs)
- Input: 16px/400, `bg-slate-700 border border-slate-600 rounded-lg px-4 py-2`
- Focus: `focus:outline-none focus:border-green-500 focus:ring-1 focus:ring-green-500`
- Error state: `border-red-500`, error text below at 14px/400 `text-red-400`
- Full width within card

### `<PrimaryButton>`
- Background: `bg-green-500 hover:bg-green-600 active:bg-green-700`
- Text: `text-white font-bold text-base` (16px/700)
- Padding: `px-6 py-3` (24px horizontal, 12px vertical)
- Border-radius: `rounded-lg`
- Width: full-width within card (`w-full`)
- Disabled state: `opacity-50 cursor-not-allowed`
- Loading state: text replaced with inline spinner (see Spinner below) + disabled

### `<Spinner>` (inline, small)
- 16×16px SVG spinning border, `border-white border-t-transparent`, animate-spin
- Used inside PrimaryButton during async operations

### `<FullScreenSpinner>`
- Fills 100vh × 100vw, `bg-slate-900`
- Centers a 40×40px spinner (`border-green-500 border-t-transparent border-4`)
- Shown while `onAuthStateChange` resolves on initial page load (D-10)

### `<CrossLink>`
- 14px/400, `text-slate-400`
- Inline `<a>` or `<Link>`: `text-green-400 hover:text-green-300 underline`
- Placed below PrimaryButton with 16px (md) top margin

### `<SuccessBanner>`
- `bg-green-900/50 border border-green-700 rounded-lg px-4 py-3`
- Text: 14px/400, `text-green-300`
- Shown on `/login` after password reset redirect (D-04)

### `<ErrorBanner>`
- `bg-red-900/50 border border-red-700 rounded-lg px-4 py-3`
- Text: 14px/400, `text-red-300`
- Used for form-level errors (network failures, duplicate username, invalid credentials)

## Page Layouts

### `/login` — LoginPage
- No Navbar
- Full-viewport centered layout: `min-h-screen bg-slate-900 flex items-center justify-center px-4`
- Primary focal point: `<PrimaryButton>` (green CTA on dark background)
- Contains: `<AuthCard>` with heading, optional `<SuccessBanner>`, email input, password input, `<PrimaryButton>`, forgot-password link, `<CrossLink>` to `/registro`
- If `location.state.from` is set, redirect to that URL after successful login (D-09)

### `/registro` — RegistroPage
- No Navbar
- Same full-viewport centered layout as `/login`
- Contains: `<AuthCard>` with heading, username input, email input, password input, `<PrimaryButton>`, `<CrossLink>` to `/login`

### `/restablecer-contrasena` — ResetPasswordPage
- No Navbar
- Same full-viewport centered layout
- Two states: (1) "enter email" form; (2) "check your email" confirmation message
- After successful reset and re-login via Supabase magic link: redirect to `/login` with success message (D-04)

### Placeholder Home (`/` or `/bienvenido`) — WelcomePage
- Renders `<Navbar>` (authenticated variant)
- Body: `min-h-screen bg-slate-900`
- Content: centered `<div>` with "Bienvenido, [username]" in Heading type (20px/700, `text-slate-100`)
- This route is replaced by `/calendario` in Phase 2 (D-14)

### `<ProtectedRoute>` (wrapper, not a page)
- Shows `<FullScreenSpinner>` while auth resolves
- Redirects unauthenticated users to `/login` with `state={{ from: location }}` (D-02, D-09)
- Redirects authenticated users away from `/login` and `/registro` to `/calendario` (D-11, placeholder to `/bienvenido` in Phase 1)

## Accessibility Notes

- All form fields must have associated `<label>` elements (not aria-label alone)
- PrimaryButton in loading state: `aria-busy="true"`, `aria-disabled="true"`
- FullScreenSpinner: `role="status"` + `aria-label="Cargando…"`
- SuccessBanner / ErrorBanner: `role="alert"` so screen readers announce on mount
- Color contrast: `slate-100` on `slate-800` → 12.6:1 (passes AAA); `green-400` on `slate-800` → 4.8:1 (passes AA)
- Minimum tap target: 44px height on all interactive elements in the navbar
