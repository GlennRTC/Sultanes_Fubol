- "Abierta" — `text-green-400 bg-green-900/50` pill (pool is accepting bets — use accent to signal action)
- "Cerrada" — `text-slate-400 bg-slate-700` pill (deadline passed, awaiting resolution — neutral/muted)
- "Resuelta" — `text-slate-500 bg-slate-800` pill (winner declared, no further action — fully muted)

- `text-slate-500` muted text — no red. Red is reserved for error banners. A loss is a resolved outcome, not an error state.

### PoolCard

One card per bet pool. Clicking a card opens the BetModal (if pool is "Abierta" and no existing bet) or BetDetailModal (if user has placed a bet, or pool is "Cerrada"/"Resuelta").

**Container:**
- `bg-slate-800 border border-slate-700 rounded-xl p-4 mb-3 cursor-pointer hover:border-slate-500 transition-colors min-h-[80px] flex flex-col gap-2`

**Pool question:**
- `text-base text-slate-100 font-bold mb-1` — the question string (e.g., "¿Quién ganará el Grupo A?")

**Status badge + deadline row:**
- `flex items-center gap-2 text-sm text-slate-400 mb-2`
- Status pill: `inline-flex items-center rounded-full px-2 py-0.5 text-xs font-medium` + status-specific color class (see Color section above)
- Deadline (open pools only): `text-xs text-slate-400` — "Cierra: [d de mes · HH:mm zzz]" (local timezone)
- Resolved label (resolved pools): `text-xs text-slate-500` — "Resultado: [winning option label]"

**Options list (parimutuel odds display — pools with no user bet, or open pools):**
- Container: `flex flex-col gap-1`
- Each option row: `flex items-center justify-between text-sm`
  - Option label: `text-slate-300`
  - Percentage: `text-slate-400` — e.g., "62%"
  - Winning option row (resolved pool): `text-green-400 font-bold` for both label and percentage

**Bet badge (shown when user has an active or resolved bet on this pool):**
- `inline-flex items-center bg-slate-700 rounded-full px-2 py-0.5 text-xs text-green-400 mt-1`
- Open/closed pool: `"Tu apuesta: [Opción] · [N] fichas"`
- Resolved pool — win: `"Ganaste: +[N] fichas"` in `text-green-400`
- Resolved pool — loss: `"Perdiste"` in `text-slate-500`

**Realtime odds animation:**
- No animation class required. Odds percentages re-render in place when Supabase Realtime pushes a new bet. No transition needed to keep implementation simple.

### BetModal (two-step confirm — active pool, no existing bet)

Follows PredictionModal structure exactly. Three render modes: (a) editable two-step, (b) read-only (user has bet), (c) closed/resolved (no action).

**Overlay:** `fixed inset-0 bg-black/60 flex items-center justify-center z-50 px-4`

**Modal panel:** `relative bg-slate-800 border border-slate-700 rounded-xl p-6 w-full max-w-[480px] mx-auto`

**Modal header:**
- Pool question: `text-xl font-bold text-slate-100 mb-1 pr-10` (pr-10 clears the close button)
- Deadline: `text-sm text-slate-400 mb-4` — "Cierra: [d de mes · HH:mm zzz]"

**Step 1 — Option selection + amount input:**

Option selector (binary: 2 buttons; multiple_exclusive: 3–4 buttons):
- Unselected option: `w-full text-left px-4 py-3 rounded-lg bg-slate-700 border border-slate-600 text-sm text-slate-300 hover:border-slate-500 min-h-[44px] mb-2 transition-colors`
- Selected option: `w-full text-left px-4 py-3 rounded-lg bg-slate-700 border border-green-500 text-sm text-slate-100 font-bold min-h-[44px] mb-2`
- Options list container: `flex flex-col gap-2 mb-4`

Live odds display per option (shown inside or below each option button):
- `text-xs text-slate-400` — e.g., "(actual: 62%)" — updates via Realtime

Token amount input:
- Label: `text-sm text-slate-300 mb-1` — "¿Cuántas fichas apuestas?"
- Input: `w-24 bg-slate-700 border border-slate-600 rounded-lg px-3 py-2 text-center text-base text-slate-100 focus:outline-none focus:border-green-500 focus:ring-1 focus:ring-green-500`
  - `type="number" min="10" step="1"`
- Helper text below input: `text-xs text-slate-400 mt-1` — "Mín. 10 fichas"
- Input + helper wrapper: `flex flex-col mb-4`

CTA button (step 1):
- `w-full bg-green-500 hover:bg-green-600 active:bg-green-700 disabled:opacity-50 disabled:cursor-not-allowed text-white font-bold text-base px-6 py-3 rounded-lg`
- Label: "Confirmar apuesta"
- Disabled when: no option selected OR amount < 10 OR amount field empty

**Step 2 — Confirmation:**

Summary line: `text-base text-slate-100 mb-2` — "[Opción seleccionada] · [N] fichas"

Confirmation body: `text-sm text-slate-400 mb-4` — "¿Estás seguro? Esta apuesta no se puede cambiar. Se descontarán [N] fichas."

Final confirm button:
- `w-full bg-green-500 hover:bg-green-600 active:bg-green-700 disabled:opacity-50 disabled:cursor-not-allowed text-white font-bold text-base px-6 py-3 rounded-lg mb-2`
- Label: "Confirmar"
- Loading state: spinner only — `inline-block w-4 h-4 rounded-full border-2 border-white border-t-transparent animate-spin`

Cancel link: `text-sm text-slate-400 hover:text-slate-300 underline text-center block w-full` — "Cancelar" (returns to step 1)

**Read-only modal (render mode b — user has already bet):**

Shows the same pool header, then:
- Selected option row: `w-full text-left px-4 py-3 rounded-lg bg-slate-700 border border-green-500 text-sm text-slate-100 font-bold min-h-[44px]` — the option the user picked
- Token amount: `text-sm text-slate-400 mt-2 mb-4` — "Apostaste: [N] fichas"
- Win badge (if resolved and won): `text-sm text-green-400 font-bold mb-4` — "Ganaste: +[N] fichas"
- Loss badge (if resolved and lost): `text-sm text-slate-500 mb-4` — "Perdiste"
- Dismiss link: `text-sm text-slate-400 hover:text-slate-300 underline text-center block w-full` — "Cerrar"
- No CTA button rendered.

**Closed/resolved modal (render mode c — no bet placed, pool closed):**
- Status notice: `text-sm text-slate-400 mb-4` — "Esta apuesta ya no acepta fichas."
- Winning option displayed with `border-green-500` border and `text-green-400 font-bold` label (resolved only).
- Dismiss link only: "Cerrar"

**Close button (all modes):**
- `absolute top-4 right-4 text-slate-400 hover:text-slate-100 text-xl leading-none min-h-[44px] min-w-[44px] flex items-center justify-center` — "×"
- `aria-label="Cerrar"`

**Error banner (step 1 and step 2):**
- Same pattern as PredictionModal: `role="alert"` div with `bg-red-900/50 border border-red-700 rounded-lg px-4 py-3 mb-4` and `text-sm text-red-300` message text.

## Accessibility Contract

- All interactive elements: minimum 44px touch target (`min-h-[44px]`)
- BetModal: `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing to pool question heading id
- Option buttons: each is a `<button type="button">` with visible label text — no icon-only targets
- Token amount input: has a `<label>` element ("¿Cuántas fichas apuestas?")
- Status badges: rendered as `<span>` with visible text — no `aria-label` needed
- Loading confirm button: `aria-busy={true}` + `aria-disabled={true}` during RPC call
- Error banners: `role="alert"` (same pattern as PredictionModal and CalendarPage)
- Pool card (acts as button): `role="button"` + `tabIndex={0}` + `onKeyDown` for Enter/Space — same pattern as MatchCard
- Close button: `aria-label="Cerrar"`
- Escape key closes modal: `useEffect` + `document.addEventListener('keydown', ...)` (reuse PredictionModal pattern)
- "Cerradas" section: pools in this section where user has no bet are still rendered but not clickable as a CTA — card is read-only, cursor default

## Data-Driven UI Rules

These rules describe how pool data maps to visual states. The executor must implement exactly these mappings.

| Condition | Visual Output |
|-----------|--------------|
| `pool.status === 'open'` AND user has no bet | Pool card: clickable → opens step-1 BetModal; "Abierta" badge |
| `pool.status === 'open'` AND user has bet | Pool card: clickable → opens read-only BetModal; "Abierta" badge; bet badge shown |
| `pool.status === 'closed'` (deadline passed, not resolved) | Pool card: clickable → opens closed-state read-only modal; "Cerrada" badge; bet badge shown if bet exists |
| `pool.status === 'resolved'` AND user won | Pool card: "Resuelta" badge; win badge; winning option row green; card → read-only modal |
| `pool.status === 'resolved'` AND user lost | Pool card: "Resuelta" badge; loss badge; winning option row green; card → read-only modal |
| `pool.status === 'resolved'` AND user has no bet | Pool card: "Resuelta" badge; winning option row green; card → read-only modal (no bet badge) |
| Realtime bet arrives on open pool | Re-render odds percentages in place for all visible pool cards — no loading indicator needed |
| Token input value < 10 OR empty | "Confirmar apuesta" button: `opacity-50 cursor-not-allowed`, `disabled` attribute set |

**Parimutuel odds calculation (frontend display only):**
- Per option %: `Math.round((option_total_tokens / pool_total_tokens) * 100)`
- If `pool_total_tokens === 0`: show "—" for all percentages (no division by zero)
- Pool total = sum of all option totals (derived from `bets` rows via Realtime)
