  <how-to-verify>
Open your Supabase Dashboard for this project and verify ALL of the following:

**Step 1 — Table Editor: confirm all four tables exist**
- Navigate to: Dashboard → Table Editor
- Confirm these tables are listed: bet_pools, pool_options, bets, token_transactions
- If any are missing, the migration did not fully apply. Re-run supabase db push.

**Step 2 — Database → Functions: confirm both functions exist**
- Navigate to: Dashboard → Database → Functions
- Confirm "place_bet" is listed with SECURITY DEFINER
- Confirm "resolve_pool" is listed with SECURITY DEFINER
- If missing, paste the SECTION 6 and SECTION 7 blocks from 0004_bet_pools.sql into the SQL Editor and run them manually.

**Step 3 — Database → Replication (Realtime): confirm bets is published**
- Navigate to: Dashboard → Database → Replication (or "Realtime" tab)
- Find the supabase_realtime publication
- Confirm the "bets" table is listed/toggled ON
- If NOT listed, run in SQL Editor: alter publication supabase_realtime add table bets;

**Step 4 — Seed one test pool via SQL Editor**
Run the following in Dashboard → SQL Editor to create a test pool and verify place_bet works. Replace the UUIDs with generated values or use gen_random_uuid() inline:

    -- Create test pool
    insert into public.bet_pools (question, type, status, deadline)
    values (
      '¿Quién ganará el Grupo A?',
      'multiple_exclusive',
      'open',
      now() + interval '7 days'
    )
    returning id;

    -- Copy the returned pool id, then insert options:
    -- (replace <pool_id> with the id returned above)
    insert into public.pool_options (pool_id, label, position) values
      ('<pool_id>', 'México', 1),
      ('<pool_id>', 'Estados Unidos', 2),
      ('<pool_id>', 'Canadá', 3);

    -- Verify pool_option_totals view works (should return 0 rows — no bets yet):
    select * from public.pool_option_totals where pool_id = '<pool_id>';

**Step 5 — Confirm resolve_pool is NOT callable by authenticated**
In SQL Editor, confirm the function exists but is not GRANTED to authenticated by checking pg_proc:

    select proname, proacl from pg_proc where proname = 'resolve_pool';

The proacl column should NOT contain an entry with "authenticated=X" or similar grant.

Type "approved" when all five steps pass. If any step fails, describe which step and what you see.
