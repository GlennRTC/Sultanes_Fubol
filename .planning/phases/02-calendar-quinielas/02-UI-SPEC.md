## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, chip spacing |
| sm | 8px | Label-to-input gap, card inner row gap, inline element spacing |
| md | 16px | Default element spacing, form field gap, card horizontal padding |
| lg | 24px | Section padding, modal vertical padding |
| xl | 32px | Page section breaks, layout column gap |
| 2xl | 48px | Major vertical section separation |
| 3xl | 64px | Page-level top/bottom padding (unused in dense calendar view) |

Exceptions:
- Touch targets (Navbar buttons, filter dropdowns, tab triggers): minimum 44px height — `min-h-[44px]` as established by Navbar.tsx.
- Match card minimum height: 72px to accommodate two-line content at mobile widths.
- Prediction modal max-width: 480px, centered with `mx-auto`.

**Source:** Navbar.tsx uses `h-14` (56px) nav, `min-h-[44px]` on buttons. LoginPage uses `px-4 py-2` (16px/8px) on inputs, `px-6 py-3` on primary button. 8-point scale confirmed.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` | Match card secondary text, timestamps, labels, badge text, filter dropdown options |
| Label / UI | 16px | 400 (regular) | 1.5 | `text-base` | Form inputs, tab labels, modal body copy, token cost reminder, leaderboard rows |
| Heading | 20px | 700 (bold) | 1.2 | `text-xl font-bold` | Page headings ("Calendario", "Tabla de posiciones"), modal match name header |
| Display | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` | Brand "FUBOL" in Navbar only |

**Hard rules:**
- Exactly 4 sizes declared. No other font sizes introduced in Phase 2.
- Exactly 2 weights: 400 (regular) and 700 (bold). Weight 600 (semibold) is not in the existing system — do not introduce it.
- Date group headers in "Por fecha" view: `text-sm font-bold text-slate-300` (14px bold) — differentiates from match card body without adding a new size tier.

**Source:** LoginPage.tsx uses `text-sm` labels, `text-xl font-bold` heading, `text-base` button. Navbar.tsx uses `text-2xl font-bold` brand. Confirmed 14/16/20/24px scale.

## Color

| Role | Tailwind Token | Hex Approx | Usage |
|------|----------------|------------|-------|
| Dominant (60%) | `bg-slate-900` | #0f172a | Page background, modal backdrop overlay base |
| Secondary (30%) | `bg-slate-800` | #1e293b | Match cards, prediction modal, Navbar, filter bar, leaderboard table rows |
| Surface-depth | `bg-slate-700` | #334155 | Form inputs, selected tab indicator background, timezone picker dropdown |
| Border | `border-slate-700` | #334155 | Card borders, modal border, input border, section dividers |
| Border-input | `border-slate-600` | #475569 | Input field borders (unfocused) |
| Accent (10%) | `text-green-400` / `bg-green-500` | #4ade80 / #22c55e | See reserved list below |
| Destructive | `bg-red-900/50` / `border-red-700` | — | Error alert banners only |

**Accent reserved for (exhaustive list — do not apply accent elsewhere):**
1. Navbar token balance display — `text-green-400` ("Fichas: N")
2. Nav link active state — `text-green-400` underline or left border indicator
3. Primary CTA buttons ("Confirmar predicción", "Confirmar") — `bg-green-500 hover:bg-green-600 active:bg-green-700`
4. Focus ring on form inputs — `focus:border-green-500 focus:ring-1 focus:ring-green-500`
5. Prediction badge on match card after submission — `text-green-400` chip ("Tu predicción: 2-1")
6. Link hover states (crosslinks, "Cambiar" timezone link) — `text-green-400 hover:text-green-300`
7. Status badge "En vivo" — `text-green-400` or `bg-green-900/50 text-green-300` pill
8. Loading spinner in FullScreenSpinner — `border-green-500` (existing pattern, reuse unchanged)

**Status badge color assignments (all three statuses):**
- "Programado" — `text-slate-400 bg-slate-700` pill
- "En vivo" — `text-green-400 bg-green-900/50` pill
- "Finalizado" — `text-slate-500 bg-slate-800` pill (muted, match is over)

**Destructive:** `bg-red-900/50 border border-red-700 rounded-lg px-4 py-3` with `role="alert"` — error banners only. No red used on buttons in Phase 2 (no destructive actions exist in this phase).

**Success/info banner (password reset inherited pattern):** `bg-green-900/50 border border-green-700 rounded-lg px-4 py-3` — reuse for post-submission confirmation if needed.

**Source:** LoginPage.tsx, Navbar.tsx, FullScreenSpinner.tsx, index.css `body { @apply bg-slate-900 text-slate-100; }`, and 02-CONTEXT.md §Established Patterns.

## Accessibility Contract

- All interactive elements: minimum 44px touch target (`min-h-[44px]`)
- Prediction modal: `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing to match heading
- Status badges: rendered as `<span>` with visible text (not icon-only) — no `aria-label` needed
- Score inputs: each has a `<label>` — "Goles [Team name]"
- Loading buttons: `aria-busy={true}` + `aria-disabled={true}` during submission (LoginPage pattern)
- Error banners: `role="alert"` (LoginPage pattern — reuse unchanged)
- Locked modal state: inputs have `disabled` attribute and `aria-disabled="true"`
- Timezone picker: when rendered as modal, trap focus within picker; close on Escape
- Filter dropdowns: native `<select>` elements (accessible without custom ARIA)
