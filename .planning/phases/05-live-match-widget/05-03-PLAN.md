Import at top: createClient from 'https://esm.sh/@supabase/supabase-js@2'

The function must NOT verify JWT (called by cron, per D-07). Do not check Authorization header at all.

ENV vars used (all available automatically in Edge Functions):
- Deno.env.get('SUPABASE_URL')
- Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')
- Deno.env.get('FOOTBALL_DATA_API_KEY')  ← set manually as a Supabase secret

The function body must implement these steps in order:

STEP 1 — Build today's date range for the API call.
  Compute today in UTC: new Date().toISOString().slice(0,10)  → "YYYY-MM-DD"
  Use same value for both dateFrom and dateTo.

STEP 2 — Fetch from football-data.org (per API Reference in CONTEXT.md):
  URL: https://api.football-data.org/v4/competitions/WC/matches?dateFrom={today}&dateTo={today}
  Header: 'X-Auth-Token': Deno.env.get('FOOTBALL_DATA_API_KEY')!
  If fetch fails or response is not ok, return JSON error with status 500.
  Parse response JSON; the matches array is at response.matches.

STEP 3 — Create the service-role Supabase client:
  const db = createClient(Deno.env.get('SUPABASE_URL')!, Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!)

STEP 4 — Load today's DB matches.
  Query: db.from('matches').select('id, home_team, away_team, match_datetime, status, home_score, away_score, external_match_id').order('match_datetime')
  Filter: only matches whose match_datetime falls on today UTC (use gte/lte with today+"T00:00:00Z" and today+"T23:59:59Z").

STEP 5 — Status mapping (per CONTEXT.md "Status Mapping"):
  Map API status values to DB status:
  - 'IN_PLAY' | 'PAUSED'  → 'live'
  - 'FINISHED'            → 'finished'
  - 'SCHEDULED' | 'TIMED' → null  (no update needed for scheduled)

STEP 6 — Team name normalization helper function (per D-05).
  Function normalize(name: string): string
    - Lowercase
    - Replace accented chars: á→a, é→e, í→i, ó→o, ú→u, ñ→n, ü→u (cover both NFD and direct)
    - Trim whitespace
  Alias map (static object) to handle known football-data.org vs our DB mismatches:
    'korea republic' → 'corea del sur'
    'iran' → 'iran'
    'united states' → 'estados unidos'
    'mexico' → 'mexico'  (our DB has 'México', normalize strips accent so both → 'mexico')
    'saudi arabia' → 'arabia saudita'
    'south africa' → 'sudáfrica'  (normalize will strip accent in DB side too)
  Apply alias map after normalizing: if normalized name is a key, return the alias value.
  This handles the known cases; unknown teams pass through normalized.

STEP 7 — Match-and-update loop.
  Track counters: updated = 0, skipped = 0, unmatched = 0.
  For each API match:
    a. Map API status → DB status (step 5). If null (SCHEDULED/TIMED), skip — no write needed.
    b. Find corresponding DB match:
       - If apiMatch.id (the integer) matches a DB row's external_match_id → fast path (exact match)
       - Else slow path: find DB match where normalize(dbMatch.home_team) === normalize(apiMatch.homeTeam.name) AND normalize(dbMatch.away_team) === normalize(apiMatch.awayTeam.name) AND dbMatch.match_datetime starts with today
    c. If no DB match found: increment unmatched, continue (log to console).
    d. Determine new values:
       - newStatus: mapped DB status ('live' or 'finished')
       - newHomeScore: apiMatch.score.fullTime.home (null if not FINISHED)
       - newAwayScore: apiMatch.score.fullTime.away (null if not FINISHED)
       - newExternalId: apiMatch.id (integer)
    e. Skip write if dbMatch.status === newStatus AND dbMatch.home_score === newHomeScore AND dbMatch.away_score === newAwayScore AND dbMatch.external_match_id === newExternalId (nothing changed).
    f. Otherwise: db.from('matches').update({ status: newStatus, home_score: newHomeScore, away_score: newAwayScore, external_match_id: newExternalId }).eq('id', dbMatch.id)
       On error: log the error, continue loop (do not abort entire sync).
    g. Increment updated counter.

STEP 8 — Return JSON summary:
  { ok: true, date: today, apiMatchCount: apiMatches.length, updated, skipped, unmatched }
  Status 200.

The entire function must be wrapped in a try/catch at the Deno.serve level; on unhandled error return { ok: false, error: err.message } with status 500.</action>
