    PART A — Deploy the Edge Function

    Step 1 — Deploy with --no-verify-jwt flag (per D-07):
      supabase functions deploy sync-live-scores --no-verify-jwt

    Step 2 — Set the API key secret (replace YOUR_KEY with your football-data.org key):
      supabase secrets set FOOTBALL_DATA_API_KEY=YOUR_KEY

    Step 3 — Note your Edge Function URL. It follows this pattern:
      https://{your-project-ref}.supabase.co/functions/v1/sync-live-scores
    Find your project ref in Supabase Dashboard → Project Settings → General.

    Step 4 — Test the function manually with curl:
      curl -X POST "https://{your-project-ref}.supabase.co/functions/v1/sync-live-scores"
    Expected response shape: {"ok":true,"date":"YYYY-MM-DD","apiMatchCount":N,"updated":N,"skipped":N,"unmatched":N}
    If you see {"ok":false,...} inspect the error field to diagnose.

    PART B — Set up cron-job.org (per D-08: every 2 minutes)

    Step 5 — Go to https://cron-job.org and log in (free account).

    Step 6 — Create a new cron job:
      - Title: fubol-sync-live-scores
      - URL: https://{your-project-ref}.supabase.co/functions/v1/sync-live-scores
      - Execution schedule: Every 2 minutes (use custom: */2 * * * *)
      - Request method: POST
      - Headers: none needed (no JWT required per D-07)

    Step 7 — Save and enable the job. Wait 2–4 minutes, then check "Execution history" in cron-job.org dashboard to confirm successful executions (HTTP 200).

    PART C — Deploy frontend

    Step 8 — Deploy the updated frontend to Netlify:
      git add src/components/TodayMatchesWidget.tsx src/pages/CalendarPage.tsx src/types/index.ts
      git commit -m "feat(phase-5): add TodayMatchesWidget with Realtime live scores"
      git push origin main
    Netlify auto-deploys on push. Wait for build to complete (~2 min).

    PART D — End-to-end verification

    Step 9 — Open CalendarPage in the browser. Verify:
      a. "Partidos de hoy" widget appears at the top IF there are matches today. If today is June 12, 2026 (opening day) matches should be present.
      b. Widget is hidden when there are no matches today (verify by checking a future date with no matches — this can be skipped if today has matches).
      c. Collapse/expand toggle works. Refresh the page — widget should restore to your last toggle state.

    Step 10 — Verify Realtime updates (if a match is live or about to go live):
      a. Keep CalendarPage open in browser.
      b. Wait for cron to fire (check cron-job.org history for next execution).

      d. If no match is live right now, you can simulate: in Supabase Dashboard → Table Editor → matches, manually set a match status to 'live'. Widget should update within a few seconds.

    Step 11 — Security check: open browser DevTools → Network tab. Confirm no request goes to football-data.org from the browser. All football-data.org requests should come from the Edge Function only.

    Step 12 — Mobile check at 375px: open DevTools → toggle device toolbar → set to 375px width. Visit CalendarPage. Confirm widget cards scroll horizontally without causing body scroll.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cron-job.org → Edge Function | Unauthenticated POST; function only writes match scores (display-only, no financial impact per D-04) |
| Edge Function → football-data.org | Outbound API call with secret key; key set as Supabase secret, not in source |
| Browser → Supabase Realtime | Anon-key websocket; only reads match updates already in the public matches table |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Information Disclosure | FOOTBALL_DATA_API_KEY in production | mitigate | Set via supabase secrets set, verified not present in network traffic (Step 11 above) |
| T-05-11 | Spoofing | cron-job.org caller | accept | Function is data-only display pipeline; no authentication needed per D-07; worst case: attacker triggers a sync (reads same public API, updates same public scores) |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan |
</threat_model>
